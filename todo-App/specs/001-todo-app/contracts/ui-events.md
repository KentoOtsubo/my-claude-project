# Contract: UIイベントと状態管理

**Type**: UI Event Contract | **Phase 1 Output** | **Date**: 2026-06-07 | **Plan**: [../plan.md](../plan.md)

## UIコンポーネントと操作

| コンポーネント | トリガー | 処理 | 根拠 |
|----------------|----------|------|------|
| 新規作成ボタン | click | 新規作成モーダルを開く | FR-001, US1-1 |
| 作成モーダル - 作成ボタン | click / Enterキー | タスクを作成してモーダルを閉じる | FR-001, US1-2 |
| 作成モーダル - キャンセルボタン | click / Escキー | モーダルを閉じる（タスク作成なし） | US1-5 |
| タスクアイテム - 完了チェックボックス | click | 完了状態をトグルする | FR-003, US2-1, US2-2 |
| タスクアイテム - 編集ボタン | click | 編集モーダルを開く（現タイトルを表示） | FR-005, US3-1 |
| 編集モーダル - 保存ボタン | click / Enterキー | タイトルを更新してモーダルを閉じる | FR-005, US3-2 |
| 編集モーダル - キャンセルボタン | click / Escキー | モーダルを閉じる（変更なし） | US3-4 |
| タスクアイテム - 削除ボタン | click | 確認ダイアログを表示する | FR-010, US4-1 |
| 確認ダイアログ - 確認ボタン | click / Enterキー | タスクを削除する | FR-007, US4-2 |
| 確認ダイアログ - キャンセルボタン | click / Escキー | ダイアログを閉じる（削除なし） | US4-3 |

## アプリケーション状態

```javascript
// app.jsが管理するメモリ上の状態
const state = {
  tasks: [],              // Task[]  現在のタスク一覧（UIの表示元）
  activeModal: null,      // 'create' | 'edit' | 'confirm-delete' | null
  editingTaskId: null,    // number | null  編集中のタスクID
  deletingTaskId: null,   // number | null  削除確認中のタスクID
};
```

## レンダリング契約

- `renderTasks(tasks)`を呼び出すとタスク一覧全体を再描画する（差分更新なし）
- 未完了タスクと完了済みタスクは視覚的に分けて表示する（FR-002, FR-004）
  - 未完了タスク: 通常スタイル、上部グループ
  - 完了済みタスク: 取り消し線・グレーアウト、下部グループ
- タスク0件の場合はエンプティステートメッセージを表示する（FR-009）
- タイトルが長い場合はUIで省略表示するが、データは全文保存する（edge case）

## バリデーション表示

| エラー条件 | UIの表示 |
|------------|----------|
| タイトルが空または空白のみ | モーダル内にエラーメッセージを表示してモーダルを開いたまま維持（FR-006, US1-4, US3-3） |

## キーボード操作（FR-011）

| キー | 状況 | 動作 |
|------|------|------|
| Tab | 全画面 | フォーカスを次の要素へ移動 |
| Enter | ボタンにフォーカス時 | ボタンのclick操作を実行 |
| Esc | モーダル/ダイアログ表示中 | モーダル/ダイアログを閉じる |

モーダルが開いた際は、入力フィールドに自動でフォーカスを移動する。
